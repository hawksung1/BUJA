## 업데이트 시점

### 작업 완료 직후
작업이 완료되고 커밋하기 전에 다음을 확인:
1. 디렉토리 구조 변경 여부
2. 새로운 파일/모듈 추가 여부
3. 아키텍처 변경 여부
4. 개발 규칙 변경 여부

### 업데이트가 필요한 경우
- ✅ 새로운 디렉토리 생성
- ✅ 새로운 주요 모듈/서비스 추가
- ✅ 설정 파일 추가/변경
- ✅ 의존성 추가/변경
- ✅ 아키텍처 패턴 변경
- ✅ 개발 규칙/컨벤션 변경
- ✅ 테스트 전략 변경

---

## 업데이트 체크리스트

### 1. 디렉토리/파일 변경 시

#### 새로운 디렉토리 추가
```
작업: src/agents/ 디렉토리 생성
→ ARCHITECTURE.md의 "모듈 구조" 섹션 업데이트
→ CURSOR_GUIDE.md의 "파일 구조" 섹션 업데이트
```

#### 새로운 주요 파일 추가
```
작업: config/settings.py 파일 생성
→ CURSOR_GUIDE.md의 "주요 파일 위치" 테이블 업데이트
→ DEVELOPMENT.md의 "설정 관리" 섹션 추가/업데이트
```

### 2. 기능/모듈 추가 시

#### 새로운 서비스 추가
```
작업: src/services/portfolio_service.py 생성
→ ARCHITECTURE.md의 "주요 컴포넌트" 섹션에 추가
→ ARCHITECTURE.md의 "모듈 구조" 섹션 업데이트
```

#### 새로운 API 추가
```
작업: REST API 엔드포인트 추가
→ ARCHITECTURE.md의 "데이터 흐름" 섹션 업데이트
→ DESIGN.md 참조 (API 설계 문서)
```

### 3. 아키텍처 변경 시

#### 레이어 구조 변경
```
작업: 새로운 레이어 추가 (예: Cache Layer)
→ ARCHITECTURE.md의 "전체 아키텍처" 다이어그램 업데이트
→ ARCHITECTURE.md의 "계층 구조" 섹션 업데이트
```

#### 데이터 흐름 변경
```
작업: 새로운 데이터 흐름 추가
→ ARCHITECTURE.md의 "데이터 흐름" 섹션에 다이어그램 추가
→ 관련 서비스 설명 업데이트
```

### 4. 개발 규칙 변경 시

#### 코드 스타일 변경
```
작업: Black 대신 Ruff 사용으로 변경
→ DEVELOPMENT.md의 "코드 스타일" 섹션 업데이트
→ pyproject.toml 설정 반영
```

#### 테스트 전략 변경
```
작업: pytest 대신 unittest 사용
→ DEVELOPMENT.md의 "테스트 가이드" 섹션 업데이트
→ 테스트 구조 설명 업데이트
```

## 가이드별 업데이트 가이드

### `.cursor/CURSOR_GUIDE.md`

#### 업데이트가 필요한 경우
- 새로운 주요 디렉토리 추가

- 프로젝트 구조 변경

1. **"빠른 참조" → "주요 파일 위치"** 테이블
2. **"파일 구조"** 섹션
3. **"참고 문서"** 섹션

```markdown

| 파일/디렉토리 | 용도 | 참고 |
|-------------|------|------|
| `config/` | 설정 파일 | 환경 변수, DB 설정 |  ← 새로 추가
```

## 업데이트 예시

### 예시 1: 새로운 서비스 모듈 추가

**작업 내용**: `src/services/user_service.py` 생성

**업데이트 필요 파일**:

1. **ARCHITECTURE.md**
```markdown
### 예상 모듈 구조 (향후 구현)

```
src/
├── services/
│   ├── user_service.py        # 사용자 관리  ← 추가
│   └── portfolio_service.py
```
```

2. **ARCHITECTURE.md - 주요 컴포넌트**
```markdown
### 2. 사용자 관리
- **위치**: `src/services/user_service.py`  ← 추가
- **책임**: 계정 생성/관리, KYC 정보 관리
```

### 예시 2: 새로운 설정 파일 추가

**작업 내용**: `config/logging.py` 생성

**업데이트 필요 파일**:

1. **CURSOR_GUIDE.md**
```markdown
| `config/` | 설정 파일 | 환경 변수, DB 설정, 로깅 설정 |  ← 업데이트
```

2. **DEVELOPMENT.md**
```markdown
### 로깅 설정
- 설정 파일: `config/logging.py`  ← 추가
- 로그 레벨: DEBUG, INFO, WARNING, ERROR
```

### 예시 3: 테스트 전략 변경

**작업 내용**: pytest-cov 추가, 커버리지 목표 80% 설정

**업데이트 필요 파일**:

1. **DEVELOPMENT.md**
```markdown
### 테스트 실행
```bash
# 커버리지 확인
pytest --cov=src --cov-report=html  ← 업데이트
```

### 목표 커버리지
- **단위 테스트**: 80% 이상  ← 추가
```

## 주의사항

### ⚠️ 업데이트 시 주의점

1. **일관성 유지**
   - 모든 가이드 파일에서 동일한 용어 사용
   - 파일 경로는 실제 구조와 일치

2. **간결성 유지 및 최적화** ⚠️
   - 불필요한 상세 설명 지양
   - 핵심 정보만 포함
   - **영어 사용 권장** (더 간결함)
   - 중복 제거
   - `.cursor/rules`는 특히 최소화 유지

3. **실제 구조 반영**
   - 가이드의 구조가 실제 프로젝트 구조와 일치해야 함
   - 예상 구조와 현재 구조 구분 명확히

4. **참조 링크 확인**
   - 다른 가이드 파일 참조 시 링크가 유효한지 확인
   - 문서 간 일관성 유지
