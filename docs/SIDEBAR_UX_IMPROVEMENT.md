# 사이드바 UX 개선 문서

## 개요
사용자 인증 상태와 온보딩 상태에 따라 동적으로 메뉴를 표시하는 공통 사이드바 컴포넌트를 구현했습니다.

## 사용자 시나리오

### 시나리오 1: 비로그인 사용자
**상황**: 사용자가 로그인하지 않은 상태

**표시되는 메뉴**:
- 🔐 Login
- 📝 Register

**표시되지 않는 메뉴**:
- 모든 인증이 필요한 메뉴 (Agent Chat, Dashboard, Profile 등)

**UX 목표**:
- 간단하고 명확한 네비게이션
- 로그인 유도 메시지 표시

---

### 시나리오 2: 로그인했지만 온보딩 미완료
**상황**: 사용자가 로그인했지만 프로필, 투자 성향, 재무 상황 중 하나 이상이 누락된 상태

**표시되는 메뉴**:
- 👋 온보딩 (필수) - **강조 표시**
  - 경고 아이콘과 함께 "온보딩 미완료" 메시지
  - 누락된 항목 목록 표시 (예: "필요: 프로필, 투자 성향")
- 🔒 주요 기능 (온보딩 후 이용 가능)
  - 🤖 Agent Chat (비활성화)
  - 📊 Dashboard (비활성화)
  - "온보딩 완료 후 이용 가능합니다" 안내 메시지

**표시되는 설정 메뉴**:
- 👤 Profile
- 📈 Investment Preference
- 🔧 MCP Tools

**UX 목표**:
- 온보딩의 중요성 강조
- 다음 단계를 명확히 안내
- 주요 기능은 비활성화하여 혼란 방지

---

### 시나리오 3: 로그인 + 온보딩 완료
**상황**: 사용자가 로그인했고 모든 필수 온보딩 정보를 입력한 상태

**표시되는 메뉴**:
- ✅ 온보딩 완료 상태 표시
- 📋 메인 메뉴
  - 🤖 Agent Chat (활성화)
  - 📊 Dashboard (활성화)
  - 👋 온보딩 (재방문 가능)
- ⚙️ 설정
  - 👤 Profile
  - 📈 Investment Preference
  - 🔧 MCP Tools
- 🚪 Logout 버튼

**UX 목표**:
- 모든 기능에 접근 가능
- 사용자 이메일 표시로 개인화된 경험 제공
- 명확한 로그아웃 옵션

---

## 구현 세부사항

### 공통 사이드바 컴포넌트
**위치**: `src/utils/sidebar.py`

**주요 함수**:
- `render_sidebar()`: 메인 렌더링 함수
- `_render_unauthenticated_sidebar()`: 비로그인 사용자용
- `_render_authenticated_sidebar()`: 로그인 사용자용
- `_check_onboarding_status()`: 온보딩 상태 확인

### 온보딩 완료 조건
다음 세 가지가 모두 충족되어야 온보딩 완료로 간주:
1. ✅ 사용자 프로필 (UserProfile)
2. ✅ 투자 성향 (InvestmentPreference)
3. ✅ 재무 상황 (FinancialSituation)

**선택사항**:
- 재무 목표 (FinancialGoal) - 온보딩 완료 조건에 포함되지 않음

### 페이지별 적용
다음 페이지들에 공통 사이드바가 적용되었습니다:
- ✅ `app.py` (메인 페이지)
- ✅ `pages/login.py`
- ✅ `pages/register.py`
- ✅ `pages/agent_chat.py` (추가 채팅 설정 포함)
- ✅ `pages/dashboard.py`
- ✅ `pages/profile.py`
- ✅ `pages/investment_preference.py`
- ✅ `pages/mcp_tools.py`
- ✅ `pages/onboarding.py`

## UX 개선 효과

### Before (개선 전)
- ❌ 로그인 여부와 관계없이 모든 메뉴 표시
- ❌ 온보딩 상태를 고려하지 않음
- ❌ 일관성 없는 사이드바 구성
- ❌ 사용자가 다음 단계를 알기 어려움

### After (개선 후)
- ✅ 로그인 상태에 따라 메뉴 동적 표시
- ✅ 온보딩 상태에 따른 명확한 안내
- ✅ 모든 페이지에서 일관된 사이드바
- ✅ 사용자 여정에 맞는 단계별 안내

## 테스트 시나리오

### 테스트 1: 비로그인 상태
1. 로그아웃 상태에서 앱 접속
2. 사이드바에 Login, Register만 표시되는지 확인
3. 다른 메뉴가 보이지 않는지 확인

### 테스트 2: 로그인 + 온보딩 미완료
1. 새 계정으로 로그인
2. 사이드바에 "온보딩 미완료" 경고 표시 확인
3. "온보딩 (필수)" 메뉴가 강조되어 있는지 확인
4. Agent Chat, Dashboard가 비활성화되어 있는지 확인
5. 누락된 항목 목록이 표시되는지 확인

### 테스트 3: 로그인 + 온보딩 완료
1. 온보딩을 완료한 계정으로 로그인
2. 사이드바에 "온보딩 완료" 상태 표시 확인
3. 모든 메뉴가 활성화되어 있는지 확인
4. 사용자 이메일이 표시되는지 확인

### 테스트 4: 페이지 간 일관성
1. 로그인 상태에서 여러 페이지 이동
2. 모든 페이지에서 사이드바가 일관되게 표시되는지 확인
3. 현재 페이지가 하이라이트되는지 확인

## 향후 개선 사항

1. **접근성 개선**
   - 키보드 네비게이션 지원
   - 스크린 리더 호환성

2. **개인화**
   - 사용자 선호도에 따른 메뉴 순서 조정
   - 최근 방문 페이지 표시

3. **알림 통합**
   - 중요한 알림이 있을 때 사이드바에 배지 표시
   - 온보딩 진행률 시각화

4. **반응형 디자인**
   - 모바일 환경에서 사이드바 최적화
   - 접기/펼치기 애니메이션
