# Analyze Skill

## 사용법

```
/analyze [분석할 기능 또는 도메인 설명]
```

## 동작 방식

1. `analyst` 에이전트를 서브에이전트로 실행
2. 현재 코드베이스 구조 파악
3. 요청된 기능을 데이터 모델 + UI 구조 + 구현 태스크로 분해
4. 결과를 `docs/specs/` 디렉터리에 마크다운으로 저장 (선택)

## 출력

- 데이터 모델 설계
- Streamlit 페이지/컴포넌트 구조
- Builder 에이전트용 구현 태스크 목록
- QA 에이전트용 테스트 기준
